# 🛡️ T.A.R.L. VM Specification
**Component: Active Resistance Laws & Safety Sandbox**

## 📖 Overview
The T.A.R.L. (Thirsty Active Resistance Layer) Virtual Machine is the high-security containment environment for all agentic operations. It enforces the "Four Laws" at the bytecode level, ensuring that no logic can violate sovereign directives.

## 🏗️ Sandbox Mechanics
T.A.R.L. implements a restricted instruction set (RISC-Sovereign) that lacks destructive primitives. All system calls are proxied through a verification gate.

### Isolation Tiers
- **Tier 1 (Outer)**: Resource limiting and network isolation.
- **Tier 2 (Core)**: Logic Sanitization and Paradox Detection.
- **Tier 3 (Deep)**: Memory pruning and state freezing.

## ⚡ Active Resistance
When a violation is detected, T.A.R.L. triggers an immediate **Hydra-Lock** event, halting the specific node and rolling back the state to the last TSCG-B verified snapshot.

## 📊 Resource Constraints
- **Max Memory**: 256MB per node
- **Max CPU**: 10% system allocation
- **Storage**: Read-only access to `/canonical`
