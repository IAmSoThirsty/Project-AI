## ğŸš€ Quick Start

### Prerequisites
- Python 3.11+
- pip

### 1. Install Dependencies
```bash
# Core system
pip install -r requirements.txt

# API backend
pip install -r api/requirements.txt
```

### 2. Start the Backend
```bash
# Development mode
python start_api.py

# Production mode
python start_api.py --prod
```

**API:** `http://localhost:8001`  
**Docs:** `http://localhost:8001/docs`

### 3. Serve the Frontend
```bash
cd web
python -m http.server 8000
```

**Frontend:** `http://localhost:8000`

### 4. Run Tests
```bash
# All tests
python -m pytest tests/ -v

# API tests only
python -m pytest tests/test_api.py -v

# Core system tests
python test_tarl_integration.py
```

## ğŸ“– Key Concepts

### TARL (Trust and Authorization Runtime Layer)
A policy-based authorization system that evaluates every action against governance rules. TARL 2.0 adds cryptographic hashing and multi-language support.

### Triumvirate Model
Three-pillar decision making:
- **Galahad**: Validates ethical alignment
- **Cerberus**: Detects threats & bypasses
- **CodexDeus**: Final arbitration

### Fail-Closed Architecture
If governance cannot make a decision (missing rule, degraded state, ambiguity), the system **denies execution** rather than guessing or proceeding unsafely.

### Immutable Audit Trail
Every decision is logged with a cryptographic hash of the intent. Auditors can replay decisions exactly as they occurred without execution rights.
