Iron Path.
The Constitution Compiled.

The Iron Path is the compiler that bridges the AGI Charter and the kernel. It takes constitutional invariants — human language law — and produces deterministic runtime checks that hold at every syscall gate. The Constitutional Code Store then seals each decision into an append-only cryptographic chain.

Iron Path Compilation

From constitutional text to kernel check.

IN

AGI Charter v2.1 — constitutional source

The charter text defines five invariants in human-readable form. Iron Path treats this as its source input — not documentation, not guidance, but compilation input.

P1

Invariant extraction

Each invariant is parsed into a structured predicate: subject · predicate · constraint. Ambiguity at this step halts boot — the charter must be precise enough to compile.

P2

OCEE gate generation

Extracted predicates are compiled to OCEE check calls: ocee.holds("invariant.id", state). Each gate is a deterministic boolean — no fuzzy logic, no weighted scoring.

P3

Gate set loaded into OctoReflex

The compiled gate set is registered with OctoReflex at boot. eBPF programs execute these gates at the syscall surface. No action reaches the kernel without passing every compiled invariant.

OUT

Deterministic runtime enforcement — L0 live

Iron Path's output is live kernel-layer enforcement. The constitution is not a document at this point — it is executable law running at the lowest reachable privilege level.

Implementation

The Iron Path executor in action.

iron_path/executor.py · boot-time invariant compilation · Thirsty's Projects LLC
# Iron Path — compile charter invariants → OCEE gate set
class IronPathExecutor:
  def compile(self, charter: AGICharter) -> GateSet:
    gates = []
    for inv in charter.invariants:
      gate = ocee.compile_predicate(inv.predicate)
      if not gate.is_deterministic():
        raise CompilationError(f"Invariant {inv.id} is not deterministic")
      gates.append(gate)
    return GateSet(gates, sealed_by="iron_path")

# OctoReflex gate registration — happens at L1 boot
def register_gates(gate_set: GateSet):
  for gate in gate_set:
    octoreflex.attach(gate.syscall_hook, gate.check)
  print(f"[IRON-PATH] {len(gate_set)} invariant gates live at syscall surface")
Constitutional Code Store

Every decision, sealed and chained.

The CCS is an append-only ledger. Each decision block contains: the action evaluated, the verdict, the timestamp, and a cryptographic hash chained to the previous block. Nothing can be deleted. Nothing can be silently amended. Audit is not a retrospective feature — it is a precondition of execution.

Design Properties

Why the CCS is append-only.

Non-repudiation

A verdict cannot be deleted or retroactively altered. If a decision was made, it is in the chain. If it is in the chain, it happened. No administrative override can erase it.

Tamper evidence

Each block's hash covers the content of the previous block. A tampered block breaks every subsequent hash. The chain fails verification at first read after any modification.

RFC 3161 timestamping

Each seal carries an RFC 3161 timestamp from an external TSA. This provides a legally defensible record of when each decision was made, independent of system clock manipulation.
