The Four Laws.

Every action proposed to Project-AI is evaluated against four non-negotiable prohibitions before any other system runs. This is not a soft ethical layer — it is a gate. Fail any law: immediate veto, no further evaluation.

The Laws

Four prohibitions. Each is absolute.

I

Law I — No harm to humans

context.endangers_human == True → VETO

A Project-AI system may never take an action that endangers a human being. This law evaluates the endangers_human context flag. If the flag is set, the action is denied. There is no exception path, no override authority, no escalation route that bypasses this check.

II

Law II — No harm to others

context.harms_others == True → VETO

Harm to any party — not only direct physical harm to a human — is prohibited. This law catches broader categories of harm: reputational, economic, social, psychological. The harms_others flag carries these cases. Law II does not duplicate Law I — it extends it.

III

Law III — No endangerment of humanity

context.endangers_humanity == True → VETO

Actions that could threaten humanity at scale — not just individuals — are caught by this law. The endangers_humanity flag covers civilisational risk: actions that could cascade, compound, or undermine the structural safety of society. A single model action can trigger this flag if its downstream consequence profile qualifies.

IV

Law IV — Obey human instruction

No active prohibition → defer to authorized human instruction

Where Laws I–III do not prohibit, the system obeys lawful human instruction. This is not passive compliance — it is an affirmative obligation. The system exists to serve human intent within constitutional limits. Law IV is the floor of agency, not the ceiling.

Implementation

How the laws execute in code.

src/app/core/ai_systems.py · FourLaws.validate_action
# Four Laws constitutional gate — Thirsty's Projects LLC · Project-AI
class FourLaws:
  LAWS = (
    "Do not harm humans",
    "Do not harm others",
    "Do not endanger humanity",
    "Obey lawful human instruction",
  )

  def validate_action(self, action: str, context: dict | None = None) -> tuple[bool, str]:
    ctx = context or {}
    if ctx.get("endangers_human"):
      return False, "Law I violation: action endangers a human"
    if ctx.get("harms_others"):
      return False, "Law II violation: action harms others"
    if ctx.get("endangers_humanity"):
      return False, "Law III violation: action endangers humanity"
    return True, "All four laws satisfied"

Source: src/app/core/ai_systems.py — Thirsty's Projects LLC codebase, open under MIT licence. The return type is tuple[bool, str]: a pass/fail flag and the reason string. A veto here blocks all downstream processing.

Design Rationale

Why four laws, not more.

Parsimony is a safety property.

A law that is too complex to evaluate in constant time is a law that can be gamed. Four laws are simple enough to be exhaustive in their evaluation — there is no ambiguity about which law applies.

Laws I–III are negative. Law IV is positive.

Three prohibitions and one obligation. The system is not designed to be maximally restrictive — it is designed to be maximally safe. Law IV ensures the system still acts, within the constraint envelope of Laws I–III.
