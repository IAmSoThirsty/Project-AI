Voting Mechanism

How Galahad's veto works.

1

Request enters the Four Laws gate

Before any other pillar evaluates the action, the Four Laws engine runs. Each law is checked independently. A violation at this stage produces an immediate veto — Cerberus and CodexDeus are not consulted.

2

Charter invariants are verified

If the Four Laws pass, Iron Path checks the action against the five compiled invariants from AGI Charter v2.1. These are deterministic — there is no interpretation, only pass or fail.

3

Galahad casts its Triumvirate vote

The result of the Four Laws and charter evaluation becomes Galahad's vote. A veto here blocks the action even if Cerberus and CodexDeus both permit it. Unanimity is required.

4

If allowed — the CCS seals the decision

Permitted actions are sealed into the Constitutional Code Store with cryptographic attestation. Every decision is permanently on record. Audit is a primitive, not a feature.
