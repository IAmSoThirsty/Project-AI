Boot Sequence

How the system starts. In order.

1

Constitutional Code Store loads

The CCS mounts the append-only constitutional ledger. No action can be sealed before the ledger is available.

2

Iron Path compiles invariants

The AGI Charter's invariants are compiled by Iron Path into deterministic runtime checks. Compilation must succeed; boot halts on any invariant violation.

3

OCEE loads compiled constraints

The Open Constraint Enforcement Engine receives the compiled invariant set. OCEE is the runtime that checks ocee.holds() at every gate.

4

OctoReflex attaches eBPF programs

eBPF programs attach to LSM and tracepoint hooks. The syscall intercept surface is live. No model action can reach the kernel without passing an OctoReflex gate.

5

NIRL reflex layer arms

Native Immune Reflex Layer loads its trigger set. Any violation during operation spawns new guardians automatically — the system is now exponentially defensive.

6

TARL runtime and PSIA arbitration come online

Policy-as-code rules load. The PSIA quorum engine and ledger initialise. The waterfall pipeline is ready to accept requests.

7

Triumvirate activates

All three pillars register their veto channels. The Triumvirate is locked. The system is fully governed and ready for operation.
