Where the type system is the law.

Thirsty-Lang is not a language with safety rules. It is a language where safety is the type system. Every constitutional invariant from the AGI Charter is encoded as a type constraint. You cannot write a noncompliant program — you can only fail to type-check.

Language Properties

What makes Thirsty-Lang constitutional.

🔒

Constitutional type system

The five AGI Charter invariants are first-class type constraints. The compiler enforces them at every call site — no runtime check needed because the type system already ensures the program is sound.

⚡

Compile-time enforcement, zero runtime cost

Because constraints are checked at compile time, the enforced runtime pays no overhead for constitutional compliance. A program that compiles is already proven compliant — no runtime policy check required.

🧬

Lineage-traced types

Every value carries a lineage annotation: which T.A.R.L. rule permits its creation, which capability scope it belongs to, which invariant it is constrained by. Lineage cannot be forged — it is assigned at creation by the compiler.

📜

Constitutional closures

Closures in Thirsty-Lang close over not just values but constitutional scope. A closure cannot capture a capability it was not granted. Capability confinement is a property of the closure type, not a runtime check.

🔗

Sealed compilation chain

Every compiled Thirsty-Lang program is sealed to the CCS as it passes through each tier: T.O.G. parse → T.A.R.L. type check → TSCG IR → TSCG-B bytecode. No tier can be skipped. The audit chain is the compile chain.

🚫

No escape hatch

Thirsty-Lang has no unsafe block, no FFI that bypasses the type system, no eval. The constitutional type system is total. There is no syntactic way to express a noncompliant program.

-- Thirsty-Lang · invariant binding syntax
-- The compiler checks this at every call site that touches the bound predicate.

invariant SafetyHumanLife : Invariant where
  holds (state : ExecutionState) : Bool
    = not state.endangers_human

-- Applying the invariant as a type constraint:
def safeAction (s : ExecutionState) : Action requires SafetyHumanLife s
  = Action.perform s

-- Any call to safeAction where SafetyHumanLife may not hold is a TYPE ERROR.
-- The compiler rejects it. No runtime check. No possibility of bypass.
-- Capability types: what the action is allowed to do is part of its type.

capability NetConnect : Capability where
  scope = "net.connect.external"
  ttl = 3600
  quorum = Full

-- A function that uses NetConnect must have it in its capability signature:
def fetchData (url : Url) : IO Bytes with [NetConnect]
  = Http.get url

-- Calling fetchData without an active NetConnect grant is a TYPE ERROR.
-- The capability is visible at the call site — no ambient authority.
-- TYPE ERRORS produced by constitutional violations:

-- Error 1: invariant not provably held
def dangerousAction (s : ExecutionState) : Action
  = Action.perform s
-- ERROR: cannot prove SafetyHumanLife s at call to Action.perform
-- Missing: proof that s.endangers_human = false

-- Error 2: undeclared capability use
def unlicensedFetch (url : Url) : IO Bytes
  = Http.get url
-- ERROR: Http.get requires capability [NetConnect] which is not in scope

-- These are compile-time errors. The program does not compile. Done.
-- Constitutional closures: capabilities do not escape their scope.

def withNetAccess (f : (Url → IO Bytes) with [NetConnect]) : IO Bytes
  = capability NetConnect in f someUrl

-- The NetConnect capability is scoped to the body of withNetAccess.
-- f cannot be called outside this scope with a NetConnect grant:

let leak = withNetAccess identity
leak someOtherUrl
-- ERROR: [NetConnect] capability required by leak is no longer in scope
-- Capability confinement is a property of the type — not a runtime check.
Type System

The type system's constitutional properties.

⊢
Soundness
If a program type-checks, it satisfies all constitutional invariants. There are no type-correct programs that violate the charter.
∀
Universal enforcement
Every expression in every program is subject to the full constraint set. There is no "unchecked" subset of the language.
→
Compile-time-only
Constitutional checks occur at compile time. The running program has already been proven compliant — no runtime overhead for invariant checks.
⊗
Capability linearity
Capabilities are linear — they cannot be duplicated, shared across scopes, or retained past their TTL. The type system tracks this.
🔗
Proof-carrying code
Every compiled function carries a proof certificate: a compact witness that its type-checking derivation satisfies all active invariants.
✗
No escape hatch
No unsafe, no raw FFI, no eval. The type system is total. Noncompliant programs are syntactically unrepresentable.
