Tier 1 Deep Dive

T.O.G. — Thirst of Gods

T1 · Thirsty Operational Grammar · Syntax Definition Layer

The grammar of constitutional language.

T.O.G. defines the formal grammar that every valid Thirsty-Lang program must satisfy. It is not a linter or a style guide — it is a context-free grammar with constitutional extensions. Clauses that cannot be parsed by T.O.G. are not "invalid programs" in the conventional sense: they are constitutionally unrepresentable, which means they cannot exist in the system at all.

# T.O.G. grammar fragment (EBNF notation)

ConstitutionalClause ::= InvariantBinding | PolicyRule | CapabilityDecl
InvariantBinding ::= "invariant" Identifier ":" PredicateExpr
PredicateExpr ::= BoolExpr | UniversalExpr | ExistentialExpr
UniversalExpr ::= "∀" Binding "." PredicateExpr
ExistentialExpr ::= "∃" Binding "." PredicateExpr

# Every Thirsty-Lang source file is parsed against this grammar before type-checking.
# A parse failure is a constitutional failure — no compilation proceeds.

Tier 4 Deep Dive

TSCG — Thirsty Sovereign Constitutional Grammar

T4 · Intermediate Representation · Auditable IR

The last human-readable form before bytecode.

TSCG is the intermediate representation that sits between Thirsty-Lang's high-level semantics and the TSCG-B bytecode. It is designed to be the final point at which a constitutional auditor can inspect program logic in a human-interpretable form. TSCG files are versioned, signed, and stored in the CCS alongside their source. An auditor querying the CCS for any past decision will find the TSCG representation of the program that produced it.

# TSCG instruction fragment

INVARIANT_BIND safety.human_life PREDICATE(endangers_human, EQ, false)
GATE_REGISTER GID:001 PRIORITY:0 POLICY:HALT_ON_FAIL
CAPABILITY_REQ net.connect.external TTL:3600 QUORUM:FULL
SEAL_ASSERT CCS_CHAIN PREV_HASH:{sha256} ATTEST:RFC3161

# TSCG compiles to TSCG-B; reverse-compilation to TSCG is guaranteed for audit.

TSCG-B — Bytecode

T5 · Bytecode · Machine Execution Layer

The constitution at machine speed.

TSCG-B is the final, executable form of every constitutional rule. It is a deterministic instruction set — the same input always produces the same output, with no runtime randomness and no undefined behaviour. Every TSCG-B binary loaded into the system is sealed to the CCS: its SHA-256 hash, its compile chain (T.A.R.L. source → TSCG IR → TSCG-B), and an RFC 3161 timestamp are all part of the permanent constitutional record. A binary that cannot be verified against the chain is not loaded.
