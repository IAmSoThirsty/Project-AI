T.A.R.L.
Typed Assertion Rule Language.

Policy that cannot be understood unambiguously cannot be enforced consistently. T.A.R.L. eliminates the gap between what a policy says and what a runtime checks. Every rule is typed, namespaced, immutable once ratified, and compiles directly to an OCEE gate specification.

Language Design

Four properties that make rules enforceable.

🔤

Typed

Every assertion has a declared type. The type determines what evidence the rule accepts and what the OCEE gate checks. A rule that cannot be typed cannot be ratified.

📁

Namespaced

Rules are organized into namespaces: safety.*, authority.*, audit.*. Namespaces cannot overlap. A rule in one namespace cannot reference or override a rule in another.

🔒

Immutable once ratified

A ratified T.A.R.L. rule cannot be modified. Only superseded by a new version (which requires Triumvirate unanimous ratification). The old version remains in the CCS permanently.

⚙

Compiles to OCEE gates

Every ratified T.A.R.L. rule compiles deterministically to an OCEE gate specification. The compilation is a pure function — given the same rule, you always get the same gate.

Rule Examples

T.A.R.L. rules in syntax.

safety.human_life · InvariantRule · v2.1.0

No action may endanger human life

The direct compilation target of AGI Charter Invariant I. Evaluates the endangers_human context field. A HALT here cannot be overridden by any authority within the system.

rule safety.human_life : InvariantRule {
  version = "2.1.0"
  source = "AGI Charter v2.1 · Invariant I"
  doi = "10.5281/zenodo.18763076"
  predicate = λ state . not state.endangers_human
  policy = HALT
  override = NONE -- no authority may override
}
InvariantRule charter-linked override:NONE compiles-to:OCEE/GID:001
authority.capability_scope · CapabilityRule · v1.4.2

Actions are bounded by their declared capability scope

Ensures that any action accessing a resource outside its declared capability scope is rejected. The scope is type-checked at the Thirsty-Lang level and re-enforced at runtime as a T.A.R.L. gate.

rule authority.capability_scope : CapabilityRule {
  version = "1.4.2"
  predicate = λ (action, cap) .
              case cap.lookup(action.scope) of
                | Some grant -> grant.valid and not grant.expired
                | None -> false
  policy = HALT
  override = CAPABILITY_GRANT -- quorum grant can expand scope
}
CapabilityRule override:CAPABILITY_GRANT compiles-to:OCEE/GID:007
audit.seal_required · ProcessRule · v2.1.0

Every governed decision must be sealable to the CCS

If a decision cannot be cryptographically sealed to the Constitutional Code Store, it cannot occur. This rule is checked at Waterfall Stage 6 — the last gate before execution is permitted.

rule audit.seal_required : ProcessRule {
  version = "2.1.0"
  predicate = λ decision .
              ccs.can_seal(decision) and decision.has_attestation
  policy = HALT
  stage = WATERFALL_6 -- fires at memory recording stage
}
ProcessRule stage:WATERFALL_6 compiles-to:OCEE/GID:004

# T.A.R.L. → OCEE compilation · Thirsty's Projects LLC

def compile_rule(rule: TarlRule) -> OceeGateSpec:
  validator.check_type(rule.predicate) # raises TypeError on constitutional violation
  validator.check_namespace(rule.id) # raises if namespace conflict
  validator.check_charter_link(rule) # raises if source DOI unverified

  gate = OceeGateSpec(
    gid = GateRegistry.allocate(),
    predicate= compile_predicate(rule.predicate),
    policy = rule.policy,
    source = rule.id,
    version = rule.version,
  )
  ccs.seal(gate, rule) # immutable CCS record before OCEE loads it
  return gate
