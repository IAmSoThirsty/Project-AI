The Core Thesis

Why language is governance.

"A safety constraint written in English is a reminder. A safety constraint compiled into a type system is a law. CodexDeus eliminates the gap between the two — the charter is the source, Thirsty-Lang is the compiler, and the bytecode is the enforcement. You cannot write a noncompliant program because noncompliant programs cannot be typed."

— Jeremy Karrick · Thirsty's Projects LLC · Project-AI constitutional architecture

Governance by type, not by policy document.

A policy document requires a reader who agrees to be bound by it. A type system requires nothing — a program that violates the type system simply does not compile. CodexDeus makes constitutional compliance a compile-time property, not a runtime aspiration.

Six tiers because the invariant must reach the machine.

Each tier translates one level of abstraction into the next: constitutional English → formal grammar → typed rules → shadow semantics → bytecode. The chain is unbroken. An invariant that cannot compile to bytecode cannot be enforced. An invariant that compiles cannot be violated without a type error.

Triumvirate Role

How CodexDeus votes in the Triumvirate.

1
Receive the capability request from PSIA
CodexDeus is consulted for any capability grant that involves language-level execution: spawning processes, compiling code, executing Shadow Thirst dry-runs, or loading new TARL rules.
2
Type-check the requested action against the active T.A.R.L. rule set
The action's type signature must satisfy all active TARL namespaces relevant to its capability scope. A type error at this point is a constitutional violation — CodexDeus votes NO.
3
Confirm Shadow Thirst dry-run converged
CodexDeus independently confirms that the Waterfall Stage 3 shadow execution report shows convergence. Any divergence, even minor, is a NO vote. CodexDeus does not re-run the shadow — it verifies the sealed result already in the Ledger.
4
Cast vote and seal to Ledger
The YES or NO vote, the type-check result, and the shadow confirmation are sealed together as a single signed entry. CodexDeus's vote is non-repudiable from the moment it is cast.
