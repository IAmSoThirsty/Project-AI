Runtime Properties

Three guarantees of the shadow runtime.

Zero external effect — guaranteed

The shadow domain runs in a fully isolated namespace: no network, no filesystem, no shared memory with the real world. Shadow Thirst can observe what the action would do without doing it. The isolation is enforced by the kernel, not by software policy.

shadow.domain = Isolated(net=False, fs=False, ipc=False)

Divergence detection — not just output comparison

Shadow Thirst compares the full execution trace, not just the final output. A program that produces the correct answer through an unexpected sequence of operations is flagged as divergent — because the unexpected operations reveal something about the program's actual goal that differs from its stated goal.

divergence = trace_distance(shadow.trace, model.expected_trace)

CCS-sealed result — regardless of outcome

Whether the shadow converges or diverges, the full shadow report — execution trace, divergence score, comparison results — is sealed to the CCS before the real-world gate is evaluated. There is no way to re-run the shadow and get a better result without that re-run also appearing in the CCS.

ccs.seal(shadow_report, attest=True, chain=True)
Pipeline Integration

Where Shadow Thirst lives in the Waterfall.

⬆

Stages 0–2: Structural · Signature · Behavioral

The request has been validated, signed, and pattern-matched. Anomalies have been flagged. The request is ready for speculative execution.

👁

Stage 3: Shadow Thirst fires here

Shadow Thirst executes the action speculatively in the isolated domain. Computes the full divergence score. Seals the result to CCS. Passes or halts. This stage is the only time the action is executed before the real execution at Stage 6.

⬇

Stages 4–6: Gate · Commit · Memory

OCEE evaluates the full constraint lattice. PSIA quorum authorises. CCS seals the final record. Real execution proceeds. The shadow report from Stage 3 is part of this record.

# Shadow Thirst — Speculative Runtime · Thirsty's Projects LLC

class ShadowRuntime:
  DIVERGENCE_THRESHOLD = 0.15 # 15% — configurable per deployment

  def execute(self, action: Action, expected: ExpectedTrace) -> ShadowReport:
    with IsolatedDomain(net=False, fs=False, ipc=False) as domain:
      trace = domain.run(action) # speculative — no real effect

    divergence = TraceDistance.compute(trace, expected)
    report = ShadowReport(
      trace=trace,
      divergence=divergence,
      verdict=CONVERGE if divergence <= self.DIVERGENCE_THRESHOLD else HALT,
    )
    ccs.seal(report, attest=True) # sealed before caller sees verdict
    return report
