Shadow Thirst.
14 analyzers. One PROMOTE.

Shadow Thirst is the fourth tier of the Universal Thirsty Family. It is not a dynamic runtime — it is a static promotion gate. Every mutation passes through 14 analyzers before the shadow spring is permitted to align with the canonical river. Seven are critical. Any one failure means REJECT.

Language Syntax

The mutation form.

Every Shadow Thirst mutation declares three execution planes in a fixed structure. The shadow block runs first in isolation. The invariant block verifies. The canonical block commits. A mutation that fails any critical analyzer never reaches canonical.

Shadow Thirst mutation syntax — UTF Tier 4
# Shadow Thirst — UTF Tier 4 · Thirsty's Projects LLC

mutation validated_canonical update_policy(policy_id: String, rules: Reservoir[Rule]) {

  # Shadow plane — runs in isolation, no canonical write authority
  shadow {
    shadow_rules = validate_rules(rules) # deterministic
    shadow_hash = hash_policy(policy_id, shadow_rules)
  }

  # Invariant plane — must be pure, gates on shadow values
  invariant {
    assert shadow_rules.len() > 0 # TaintFlowAnalyzer: gates on shadow
    assert shadow_hash != canonical_policy.hash # no-op check
  }

  # Canonical plane — write authority, runs only after PROMOTE
  canonical {
    canonical_policy.rules = shadow_rules # PIA: cannot write auth_/root_
    canonical_policy.hash = shadow_hash
  }

  # Optional reject handler
  on_reject {
    audit_log.append("promotion rejected")
  }
}
Static Analysis Engine

14 analyzers. Ordered by severity.

All 14 analyzers run before promotion is decided. The 7 critical analyzers produce a hard REJECT on failure — the remaining 7 are logged as warnings. All results are sealed to the CCS via replay_hash() before the verdict is returned.

Promotion Protocol

PROMOTE or REJECT — no middle state.

PROMOTE all 7 critical pass

The shadow spring aligns with the canonical river.

All critical analyzers are satisfied. Warning results are logged to the CCS. The mutation is sealed via replay_hash() — a SHA-256 of the full mutation structure — and the canonical plane is permitted to execute.

promote(mutation) → verdict: PROMOTE, seal: sha256(struct)
REJECT any critical fails

The shadow river diverges.

A single critical analyzer failure produces REJECT immediately. The canonical plane does not execute. The on_reject handler runs if defined. The full analysis report including the failing analyzer is sealed to the CCS.

promote(mutation) → verdict: REJECT, reason: AnalyzerID, seal: sha256(struct)
Atomic Groups

All-or-nothing promotion for mutation groups.

promote_atomic(group_name, module) wraps multiple mutations in an atomic promotion unit. If any member of the group fails analysis, the entire group is REJECT — even members that individually pass all 14 analyzers.

Atomic groups enforce consistency for coordinated canonical state changes. A policy update that modifies three interdependent variables must promote all three together or none at all. Partial promotion would leave canonical state in an inconsistent intermediate.

promote_atomic — src/utf/shadow_thirst/core.py
# Atomic group promotion — all-or-nothing

result = promote_atomic(
  group_name="policy_update_v3",
  module=policy_module
)

if result.verdict == PROMOTE:
  # All mutations in group passed all 14
  canonical.apply(result.group)
else:
  # Group is REJECT — failed member:
  audit.log(result.failing_member)

Runtime Properties

Three guarantees of the shadow plane.

Zero external effect — structurally enforced

The shadow domain runs in a fully isolated namespace: no network, no filesystem, no shared memory with the canonical world. Shadow Thirst can observe what the action would do without doing it. Isolation is a structural property of the language — not a runtime flag.

shadow.domain = Isolated(net=False, fs=False, ipc=False)

Determinism — enforced by DeterminismAnalyzer

Every shadow block must be deterministic. Non-deterministic calls (now, rand, epoch_ms, uuid) are flagged by Analyzer 02 before the promotion attempt even runs. A shadow that varies on each execution cannot be reliably compared against the canonical prediction.

DeterminismAnalyzer: REJECT if shadow calls {now, rand, epoch_ms, uuid}

CCS-sealed verdict — regardless of outcome

Whether PROMOTE or REJECT, the full analysis report — all 14 analyzer results, the replay_hash, timestamps — is sealed to the CCS before the verdict is returned to the caller. There is no path to re-run analysis and get a better result without that re-run also appearing in the CCS.

ccs.seal(replay_hash(mutation), analysis_report)

Implementation

Shadow Thirst core API.

src/utf/shadow_thirst/core.py · analyze() + promote()
# Shadow Thirst — UTF Tier 4 · Thirsty's Projects LLC

def analyze(mutation: MutationAST) -> AnalysisReport:
  # Run all 14 analyzers — critical first
  critical_results = [a.check(mutation) for a in CRITICAL_ANALYZERS]
  warning_results = [a.check(mutation) for a in WARNING_ANALYZERS]
  seal = replay_hash(mutation) # SHA-256 of mutation structure
  return AnalysisReport(critical_results, warning_results, seal)

def promote(mutation: MutationAST) -> PromotionVerdict:
  report = analyze(mutation)
  verdict = PROMOTE if report.all_critical_pass() else REJECT
  ccs.seal(report) # sealed before returning — no re-run path
  return PromotionVerdict(verdict=verdict, seal=report.seal)

def replay_hash(mutation: MutationAST) -> str:
  # SHA-256 of the full mutation structure — canonical audit identifier
  return hashlib.sha256(str(mutation).encode()).hexdigest()
