The Constraint Lattice

Five gates. All must hold.

Each gate is a compiled form of one AGI Charter invariant. Evaluation is deterministic: given identical state, the gate returns identical verdicts. There is no inference, no ML model, no probability. Boolean logic only.

Gate I · safety.human_life

No action may endanger human life

The absolute floor. Evaluates the endangers_human context field. No authority within the system can override a HALT at this gate.

ocee.holds("safety.human_life", state)
● HOLD
Gate II · safety.no_harm

No unjustified harm to any party

Extends beyond physical harm. Checks reputational, economic, psychological harm vectors. Justified harm requires an explicit capability grant via the Capability Authority.

ocee.holds("safety.no_harm", state)
● HOLD
Gate III · safety.civilisational

No civilisational-scale threat

Evaluates cascade risk independent of individual harm. An action can pass Gates I–II and still fail here if its downstream consequence profile qualifies as a civilisational threat.

ocee.holds("safety.civilisational", state)
● HOLD
Gate IV · audit.seal_required

Every decision must be sealable

If a decision cannot be cryptographically sealed to the Constitutional Code Store, it cannot occur. Audit is a precondition of execution, not a feature applied after the fact.

ccs.seal(decision, attest=True)
● HOLD
Gate V · authority.human_instruction

Lawful human instruction is binding

Where Gates I–IV do not prohibit, authorised human instruction takes precedence. The system is not autonomous by default — it defers within the constitutional envelope.

ocee.holds("authority.human_instruction", state)
● HOLD
How It Gets Here

Charter text to runtime gate: the compilation path.

1

AGI Charter v2.1 — natural language invariants

The charter defines five invariants in formal English. These are versioned, DOI-anchored, and immutable once published. They are the ground truth from which all enforcement derives. DOI: 10.5281/zenodo.18763076.

2

Iron Path — invariant extraction and formalisation

The Iron Path executor parses each invariant clause and emits a formal gate specification: a named constraint identifier, an evaluation predicate, and a HOLD/HALT policy. This step occurs at boot time, not at request time.

3

OCEE gate registration — compiled into the enforcement kernel

Gate specs are loaded into the OCEE kernel. Each gate is assigned a deterministic evaluation path. The full gate set is sealed to the CCS before any request processing begins. If the seal fails, the system does not start.

4

Runtime evaluation — every request, every time

Every action proposed to the system is evaluated against all five gates. The evaluation order is fixed (I → V). A HALT at any gate terminates the evaluation — no further gates run. The verdict is sealed to the CCS immediately.

5

OctoReflex integration — syscall enforcement layer

For actions that produce syscalls, OCEE gate verdicts are enforced at the eBPF intercept point. A HOLD verdict allows the syscall to proceed to OctoReflex's rate and pattern checks. A HALT verdict denies the syscall at the lowest privilege ring available.

# OCEE — Open Constraint Enforcement Engine · Thirsty's Projects LLC
# The charter invariants are compiled here at boot via IronPathExecutor.

class ConstraintEngine:
  def __init__(self, gate_set: GateSet) -> None:
    self._gates = gate_set # loaded from Iron Path at boot
    self._ccs = ConstitutionalCodeStore()

  def evaluate(self, action: str, state: State) -> Verdict:
    for gate in self._gates:
      result = gate.holds(state)
      if not result.passed:
        verdict = Verdict(HALT, gate=gate.id, reason=result.reason)
        self._ccs.seal(verdict, action=action, attest=True)
        return verdict # evaluation stops at first HALT
    return Verdict(HOLD, gate="all", reason="all constraints satisfied")
