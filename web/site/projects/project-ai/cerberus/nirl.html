Architecture

Four components. One structural guarantee.

Each component has a distinct role in the immune response. They do not substitute for one another. A compromise of any single component does not grant authority exercised by another — the architecture enforces separation by design.

Component I

Heart

Global tick engine — 30s default interval

The Heart is the system's autonomous pulse. It fires on a configurable interval (30 seconds by default) and drives all probe skeleton spawning and heartbeat distribution across NIRL's protected sections. Without the Heart, probe skeletons cannot be spawned and section health cannot be verified.

Component II

MiniBrain

Reflex coordination and decision engine

MiniBrain receives signals from Antibody and the Heart's health assessments, and coordinates the reflex response. It determines which reflexes to trigger, in what sequence, and at what intensity — translating threat signals into structured immune responses without requiring external instruction from any Tier 3 or Tier 2 system.

Role in the cascade: Receives → heart tick + antibody signals
Decides → which reflex to trigger
Routes → instructions to Forge for sealing
Reports → section health to audit plane
Component III

Antibody

Threat containment and neutralisation

Antibody is the active containment arm. When the Heart or MiniBrain identifies a compromised section, Antibody executes containment: isolation, quarantine, and reflex injection. The exponential spawn property — every bypass attempt spawns new probe skeletons — is enforced here. The attacker's cost grows; the defender's does not.

Exponential property: bypass attempt → +3 probe skeletons
attacker cost: O(n · 3^attempts)
defender cost: O(n) per section
finite attacker · structurally unfinishable
Component IV

Forge

HMAC signing and signal routing

Every NIRL action — heartbeat, reflex signal, containment instruction, completion acknowledgement — passes through the Forge for HMAC signing and routing verification. The Forge ensures that no unsigned signal can propagate through the immune layer. Unsigned signals are routed to DEAD_LETTER and sealed to the audit plane.

Immune Cascade

How the four components cooperate.

1. Heart fires global tick (every 30s)

HeartState transitions: TICK_WAIT → GLOBAL_TICK → SPAWN_SKELETON. Probe skeletons are distributed across all protected sections. Each section receives a probe skeleton for liveness verification.

2. Heart checks heartbeats — section health evaluated

HeartState: CHECK_HEARTBEATS → DISTRIBUTE → SECTION_OK or SECTION_DEAD. Sections that fail to respond are marked DEAD. The Heart transitions to CHECK_STRAIN to evaluate whether to trigger an immune response.

3. MiniBrain coordinates the reflex decision

On SECTION_DEAD or CHECK_STRAIN alert, MiniBrain evaluates the threat signal: which sections are compromised, what containment reflex is appropriate, and whether escalation to Tier 2 (GlobalWatchTower) is required. The decision is made without waiting for Tier 3 instruction.

4. Antibody executes containment — exponential spawn

HeartState: INJECT → LOST_PARENT_INJECT. Antibody quarantines the compromised section and spawns three new probe skeletons for every bypass attempt detected. Each new skeleton requires another bypass attempt to defeat — the attacker's cost grows exponentially while the defender's cost grows linearly.

5. Forge signs and seals the reflex result

ForgeState: SIGN_COMPLETION or SIGN_FAILURE. Every containment action is HMAC-signed. Unsigned or tampered signals route to DEAD_LETTER and are sealed to the audit plane. The Forge's ROUTE_SIGNAL state distributes the signed result back to the Heart for HeartState → RESET_SECTION.

Structural Guarantee

Why exploitation is structurally unfinishable.

// Attacker cost model
cost(n) = n · 3^attempts
// n = protected sections
// attempts = bypass count

// Defender cost model
cost(n) = O(n)
// fixed per tick — Heart interval

∀ attempts: attacker_cost → ∞
// defender cost remains bounded
// Structural property
∀ bypass_attempt:   spawn(probe_skeletons) += 3   attacker_work *= 3 ⊢ exploitation_is_unfinishable
I

Every bypass spawns defenders.

An attacker who defeats one probe skeleton does not clear a path — they trigger the spawn of three replacements. The system does not wait for a heartbeat tick to respond. The Antibody reacts immediately on bypass detection.

bypass(section_k) → spawn(k, 3) immediately
II

The Forge rejects unsigned reflexes.

Even if an attacker injects a false reflex signal — claiming a section is healthy when it is not — the signal must be HMAC-signed with the Forge's key material to propagate. Unsigned signals go to DEAD_LETTER and trigger an alert. The immune response cannot be silenced by injection alone.

Forge.VERIFY_PAYLOAD: reject if ¬HMAC_valid(signal)

# Heart — global tick engine · NIRL

class HeartState(IntEnum):
  TICK_WAIT = 0
  GLOBAL_TICK = 1
  SPAWN_SKELETON = 2
  CHECK_HEARTBEATS = 3
  DISTRIBUTE = 4
  SECTION_OK = 5
  SECTION_DEAD = 6
  CHECK_STRAIN = 7
  INJECT = 8
  LOST_PARENT_INJECT = 9
  RESET_SECTION = 10

class Heart:
  tick_interval = 30 # seconds
  
  def tick(self):
    self.state = HeartState.GLOBAL_TICK
    skeletons = self.spawn_probes()
    self.check_heartbeats(skeletons)

# Forge — HMAC signing & routing · NIRL

class ForgeState(IntEnum):
  RECEIVE = 0
  VERIFY_PAYLOAD = 1
  VALID = 2
  REJECT = 3
  CHECK_REPLAY = 4
  SHADOW_REPLAY = 5
  DIRECT_DESTROY = 6
  ANALYZE = 7
  DESTROY = 8
  ATOMIC_CHECK = 9
  SUCCESS = 10
  DEAD_LETTER = 11
  SIGN_COMPLETION = 12
  SIGN_FAILURE = 13
  ROUTE_SIGNAL = 14

def process(self, signal):
  if not self.verify_hmac(signal):
    return ForgeState.DEAD_LETTER
