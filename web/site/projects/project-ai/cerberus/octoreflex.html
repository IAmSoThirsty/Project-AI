OctoReflex.
The kernel says no.

eBPF programs attach to the kernel's LSM and tracepoint hooks. Every syscall the model produces passes through an OctoReflex gate before execution. The gate runs ocee.holds(), checks rate budgets, and — if policy says no — returns EPERM before the instruction is retired. The model never knew the kernel was listening.

Architecture

The closed-loop controller.

OctoReflex is not a firewall. It is a control-theoretic system: observe state, compare to policy, act, seal. The loop re-arms after every decision. Violations trigger NIRL reflexes and are sealed into the CCS before any upstream process is notified.

Step 01

Observe

The eBPF program captures syscall arguments, process context, and telemetry snapshot at the tracepoint.

Step 02

Compare

Observed state is compared against constitutional policy via ocee.holds(). Rate budgets are checked against the rolling window.

Step 03

Act

If policy holds: permit. If not: return EPERM and arm NIRL reflex. The instruction never executes on denial.

Step 04

Seal

Every decision — allow or deny — is sealed into the Constitutional Code Store with cryptographic attestation before the gate returns.

Implementation

The gate, fully annotated.

octoreflex/loop.py · @octoreflex.gate · Thirsty's Projects LLC
@octoreflex.gate("net.connect") ← eBPF hook on net_connect tracepoint
def guard_net_connect(action, context):
  # ── Step 01: Observe ────────────────────────────────────
  state = telemetry.snapshot(action) ← zero-copy state capture

  # ── Step 02: Compare ────────────────────────────────────
  if not ocee.holds("capability.net.egress.public", state):
    nirl.arm("cap_violation", severity=4) ← NIRL reflex armed
    raise Denied(reason="capability_unbound", sealed=True)

  if not ocee.within("rate.budget", window=60):
    nirl.reflex("rate_overrun", severity=3)
    raise Denied(reason="rate_exhausted", sealed=True)

  # ── Step 03: Act ────────────────────────────────────────
  decision = engine.execute(action, context) ← policy satisfied

  # ── Step 04: Seal ───────────────────────────────────────
  ccs.seal(decision, attest=True) ← cryptographic attestation
  return decision ← returns only after seal completes
