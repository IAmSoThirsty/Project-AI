## Archive Maintenance

### Archival Process

1. **Identification**: Documents marked as completed, superseded, or deprecated
2. **Categorization**: Assigned to appropriate subdirectory based on subject
3. **Metadata**: Date, subsystem, and reason recorded in this index
4. **Git History**: Preserved via `git mv` to maintain full history
5. **Index Update**: This file updated with new entries

### Search & Query

To find archived documents:

```bash
# Search by keyword
grep -r "keyword" docs/internal/archive/

# List all security-related archives
find docs/internal/archive/ -name "*SECURITY*" -o -name "*security*"

# List files by date (git log)
git log --follow docs/internal/archive/FILENAME.md
```

### Retention Policy

- **Completion Documents**: Retain indefinitely (historical record)
- **Status Documents**: Retain for 2 years, then evaluate
- **Session Notes**: Retain for 1 year unless referenced
- **Security Incidents**: Retain indefinitely (compliance requirement)
