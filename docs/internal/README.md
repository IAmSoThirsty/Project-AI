# Internal Documentation

This folder is reserved for legacy notes, archived summaries, internal codenames, and engineering team-only documentation.

## What Belongs Here

- **Implementation summaries** - Internal implementation notes and summaries
- **Integration summaries** - Internal integration documentation
- **Session notes** - Development session notes and progress tracking
- **Legacy documentation** - Archived or superseded documentation
- **Internal codenames** - Documentation using internal terminology (Triumvirate, Cerberus, etc.)
- **Work-in-progress** - Draft documentation not ready for external audiences
- **Technical debt tracking** - Internal technical debt and refactoring notes
- **Historical records** - Project history and evolution

## Key Subdirectories

- `archive/` - Archived and historical documentation
  - `historical-summaries/` - Historical project summaries
  - `session-notes/` - Development session notes
  - `security-incident-jan2026/` - Security incident documentation
  - `adversarial-completion/` - Adversarial testing archives

## Guidelines

- **Audience**: Internal engineering team only
- **Tone**: Can be informal, use internal jargon and codenames
- **Content**: Implementation details, internal processes, work-in-progress
- **Not for external audiences**: Do not reference these documents in public-facing materials

## Internal Codenames (OK to use here)

- **Triumvirate** - The three-part governance system
- **Cerberus** - Security sentinel agent
- **Galahad** - Ethics enforcement agent
- **Codex Deus Maximus** - Governance code repository

## Note

Documents in this folder should NOT be referenced in:
- Public documentation
- Investor materials
- External APIs or user interfaces
- Public presentations or whitepapers
