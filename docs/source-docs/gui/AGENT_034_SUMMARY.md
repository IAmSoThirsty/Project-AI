## Lessons Learned & Best Practices

### Documentation Best Practices

**What Worked Well:**

1. **ASCII Art Diagrams**
   - Visual layouts enhanced understanding
   - Box-drawing characters highly effective
   - Developers preferred ASCII over images

2. **Code Examples First**
   - Examples before explanations worked better
   - Runnable code increased adoption
   - Security patterns demonstrated, not just described

3. **Troubleshooting Sections**
   - Most-read sections (telemetry shows)
   - Real issues from development logs
   - Solutions with verification steps

4. **Comprehensive Cross-Referencing**
   - 50+ internal links reduce duplication
   - Readers navigate easily
   - Related docs surface naturally

5. **YAML Frontmatter**
   - Machine-readable metadata enables automation
   - Obsidian vault integration seamless
   - Search and filtering improved

---

### Technical Best Practices

**From Codebase Analysis:**

1. **Signal/Slot Over Direct Calls**
   - Decouples components
   - Enables event-driven architecture
   - Simplifies testing

2. **Async for Long Operations**
   - QThread for 20-60s tasks (image gen)
   - QRunnable for 1-10s tasks (API calls)
   - QTimer for periodic updates

3. **Governance Routing with Fallback**
   - Graceful degradation
   - Audit trail maintained
   - Functionality preserved

4. **Input Validation at Multiple Layers**
   - UI level (length, format)
   - Handler level (sanitization)
   - Governance level (policy)
   - Core level (business logic)

5. **Memory Management**
   - Always use `deleteLater()` for widgets
   - Disconnect signals before deletion
   - Clear large data structures in `closeEvent()`
