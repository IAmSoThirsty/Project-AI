REPAIR GROUP 2 — INCOMPLETE IMPLEMENTATION (13 failures fixed)
=============================================================================

Root cause: Planned API surface documented in test_hydra_comprehensive.py but not
yet implemented in the engine. No design conflict — straightforward additions.

  5. BaseScenario.tick()
       → calls self.evaluate_escalation()
       All concrete scenario classes inherit this; no override needed.

  6. BaseScenario.escalate()
       → advances escalation_level by 1 step (capped at LEVEL_5_COLLAPSE)
       → updates status: TRIGGERED (level >= 1 + was DORMANT), ESCALATING (>= 2),
         COLLAPSE (>= 5)
       → sets activation_time if not already set
       Each call advances exactly one level; rapid escalation tests confirm cap at 5.

  7. Hydra50Engine.get_critical_scenarios() -> list[str]
       → returns scenario_ids where escalation_level.value >=
         EscalationLevel.LEVEL_4_CASCADE_THRESHOLD.value

  8. Hydra50Engine.reset_scenario(scenario_id: str)
       → resets status to DORMANT, escalation_level to LEVEL_0_BASELINE,
         clears metrics, trigger_active flags, activation_time, suppressed,
         poisoned_recovery flags

  9. Hydra50Engine.reset_all_scenarios()
       → calls reset_scenario for every scenario_id in self.scenarios

  10. Hydra50Engine.execute_control_plane_command(plane, command, params=None) -> dict
        → routes "status" command: returns get_dashboard_state()
        → routes "override" command with params["scenario_id"] + params["action"]:
          - "suppress": sets scenario.suppressed = True
          - unknown action: returns {"error": "unknown action"}
        → unknown command: returns {"error": "unknown command"}
        → unknown plane type: returns {"plane": str(plane), "command": command,
                                        "result": "acknowledged"}
