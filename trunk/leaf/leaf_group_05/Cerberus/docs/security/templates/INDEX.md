## 🛡️ Security & Classification

### Report Classification Levels

| Level | Audience | Handling |
|-------|----------|----------|
| PUBLIC | Anyone | No restrictions |
| INTERNAL | Employees | Internal networks only |
| CONFIDENTIAL | Leadership + Security | Secure email/portal |
| RESTRICTED | Need-to-know only | Encrypted transmission |

### Handling Sensitive Reports

✅ **Secure Storage:** Encrypted drives, secure archives
✅ **Secure Transfer:** Encrypted email or secure portals
✅ **Physical Security:** Shred printed copies
✅ **Data Retention:** Follow organizational policy (typically 3-7 years)
✅ **Secure Deletion:** Permanently delete when no longer needed

## 📈 Best Practices

1. **Use actual data** - Not placeholders or estimates
2. **Include evidence** - Logs, screenshots, code samples
3. **Be specific** - Exact IPs, filenames, usernames
4. **Complete all sections** - Don't leave items blank
5. **Get timely approvals** - Don't delay sign-offs
6. **Follow classification** - Use correct security levels
7. **Archive properly** - Keep reports for compliance
8. **Review examples** - Learn from past completed reports
9. **Guardian focus** - Always analyze Guardian impact
10. **Professional quality** - Represents your organization
